# stack与单调思想使用场景
1. 核心本质：一个容器适配器，默认在deque上实现
2. 最大优势：极端的秩序。因为接口被封死，你不会犯月结修改，中间误删地错误，状态管理极其严密
3. 致命弱点：完全没有迭代器。你无法遍历一个 stack，也无法查看栈底或中间的元素。如果你想拿到栈底的元素，必须把上面的元素全部弹空销毁（或者倒腾到另一个栈里）。

## 1. 绝对使用场景
1. 需要处理嵌套上下文，或者对称匹配地场景。遇到左半边(入栈保存状态)，遇到右半边(出栈合并/消除状态)
2. 需要寻找“下一个更大/更小地元素”：单调栈的绝对主场
3. 需要将递归强行转化为迭代的场景。所有的系统递归调用底层其实就是栈。如果你要写二叉树的非递归前、中、后序遍历，或者在极深递归时为了防止栈溢出 (Stack Overflow)，你自己建一个 std::stack 就能完美模拟系统压栈的过程。

## 高频解题套路和技巧
1. 套路1 ： 单调栈
    - 原理：利用栈来维护一个"待解决问题"的集合。栈内必须存储下标(为了算距离或判定存货状态)，并且保证这些下标对应的数组严格保持单调递增或递减。当遇到一个破坏单调性的新元素时，说明栈顶元素的下一个更大或更小值出现了，进行弹栈结算
    - 典型题例：每日温度，接雨水，柱状图中的最大矩形
2. 套路2：多状态封存与嵌套解码
   - 原理：在遇到表示"进入内层"的符号(如'['或'(')时，将当前外层的累计状态(当前的乘数，当前已经拼好的前缀字符串)分别压入栈中保护起来，在遇到退出内层符号时，将内层算好的结果与栈顶的外层状态进行合并
   - 典型例题：字符串解码，基本计算器
3. 套路3：调度场算法与逆波兰表达式
   - 原理：利用栈，根据栈，根据数学运算符号的优先级，将人类阅读的"中缀表达式"转化为计算机极易执行的‘后缀表达式’。遇到数字输出，遇到符号判断优先级决定压栈或者弹栈
   - 典型例题：逆波兰表达式求值，复杂算术表达式求值

## 3. 避坑指南
1.  top() 的 UB 陷阱：与队列的 .front() 极其类似，对一个空的 stack 调用 top() 或 pop() 没有任何底层安全保护，直接触发未定义行为（大概率段错误崩溃）。对栈顶动刀子前，永远记得先判空 !st.empty()。
2.  出栈没有返回值（C++ 独有特色）：在 Java 或 Python 中，调用 pop() 会直接把弹出的元素返回给你。但在 C++ 中，pop() 的返回类型是 void！ 它只负责删，不负责给。你必须先写 x = st.top(); 把值拿出来存好，然后再调用 st.pop();。
3. 底层物理容器的极致替换：虽然 stack 默认底层是 deque，但如果你明确知道这道题只需要尾部操作，并且数据量极大（几百万），可以手动把底层换成 vector，写法是：std::stack<int, std::vector<int>> st;。这能极大增加 CPU Cache 的命中率，在极其苛刻的卡常数机试中能帮你榨干最后的性能。
