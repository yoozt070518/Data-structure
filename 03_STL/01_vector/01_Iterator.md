# 迭代器
## 本质
- 一个包装过的超级指针
- STL 的作者为了让你能用完全相同的代码去遍历 vector（底层是数组）和 list（底层是链表），特意发明了迭代器。你不管底层的物理结构是什么，只要拿到这个“超级指针”用 ++ 它就能走到下一个位置,用 * 就能拿到里面的数据。
## 核心
- begin(): 返回一个指向队伍第一个人的迭代器
- end(): 返回最后一个人后面的空位
- “左开右闭”
- 当迭代器不断++ 一旦等于end() 就说明队伍遍历完了
## 操作
1. 拿数据(解引用): *it
    - 和普通指针一模一样。it只是一个地址坐标，加上*才能拿到坐标里存的真实数据
2. 移动位置: ++it或 --it
    - 指针走到下一个人或上一个人(++it的性能更好)
4. 大跨步移动it+5
```cpp
   int main() {
    vector<int> v = {10, 20, 30};

    // 1. 极其冗长的老式类型声明（现在一般用 auto 替代）
    // vector<int>::iterator it; 
    
    // 2. 用迭代器遍历队伍
    for (auto it = v.begin(); it != v.end(); ++it) {
        
        // it 是超级指针，*it 才是里面的数字
        cout << *it << " "; 
        
        // 你甚至可以通过超级指针直接修改里面的数字
        // *it = 99; 
    }
    
    return 0;
}
```
## 迭代器失效
1. 大搬迁引起的悬空危机
    - 在容器进行扩容时 在迁移时it迭代器不会跟着迁移 此时再进行解引用就会出错
    - 在删除中间数据的时候 将数据删除后 后面的数据会立即补上 而此时直接执行it++就会漏查数据
    - ++解决办法++：erase 和 insert 函数执行完毕后，都会主动返回一个全新、完全合法的迭代器！你需要做的，就是接住它，更新你的坐标！
```cpp
vector<int> v = {10, 20, 20, 30};

for (auto it = v.begin(); it != v.end(); /* 🌟 这里千万不能写 ++it */ ) {
    if (*it == 20) {
        // 🌟 解药：erase 会返回被删元素的“下一个有效元素的坐标”。
        // 我们用这个新坐标直接覆盖旧的 it！
        it = v.erase(it); 
    } else {
        // 只有没开枪的时候，巡逻兵才自己往前走一步
        ++it; 
    }
}
```
```cpp
vector<int> v = {10, 20, 30};

for (auto it = v.begin(); it != v.end(); /* 依然留空 */ ) {
    if (*it == 20) {
        // 1. 开枪插入 99，接住新坐标 (it 指向 99)
        it = v.insert(it, 99); 
        
        // 2. 🌟 满分操作：强行跨步！
        // 跨过刚才插入的 99，再跨过原本的那个 20，直接去看下一个人 (30)
        it += 2; 
    } else {
        // 没遇到 20，正常巡逻，走一步
        ++it; 
    }
}
```
