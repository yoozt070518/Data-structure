# priority_queue
## 1. 核心与STL源码剖析
1. 核心思想
- 普通队列：先进先出(FIFO)，是绝对的时间公平
    - 出队顺序 = 进队顺序(时间驱动)
- 优先队列：只关心优先级 优先级高的先出队
    - 出队顺序 = 优先级由高到低(权重驱动)

2. STL源码参数解析
```cpp
template <
    typename T, 
    typename Container = std::vector<T>, 
    typename Compare = std::less<typename Container::value_type>
> 
class priority_queue;
```
T ：数据类型
Container : 底层容器类型
Compare：比较仿函数

## 2. API接口
1. top()
   - 内存动作：调用vector::front()
   - 时间复杂度：O(1)

2. push(val): 新贵上位(向上调整Sift Up)
   - 当有一个新元素进入优先队列时，它不能直接坐到最高优先级的位置，必须遵循"堆"的晋升规则
       - 内存动作：首先调用vector::push_back(val)，将新元素挂在物理数组的最后面(在逻辑树上，就是挂在最后一层的最末尾，保证完全二叉树的形状)
       - 向上调整(SiftUp/源码中的std::push_heap):新元素开始于其父节点(i-1)/2比较。如果新元素的优先级高于父节点，两者交换。持续向上浮动，直到遇到一个比它强的父节点，或者自己变成了根节点
       - 时间复杂度：O(logn)，树的高度是logn，最坏情况下新元素一路干掉所有上司飙升到根节点，最多发生logN次比较和内存交换
       - 空间复杂度: 平摊O(1)。除了vector扩容时触发的内存重分配，本身不需要额外空间

3. pop(): 旧王退位，权力重组(SiftDown)
    - 将优先级最大的元素移出队列，是整个数据结构中最危险，也最考验性能的操作，因为它直接破坏了堆顶
      - 内存动作：
        1. 首位交换：绝对不能直接把数组第0个元素删掉(那会导致整个数组前移，引发灾难性的O(N)内存拷贝)。STL的做法是：直接把堆顶(第0个)和堆尾进行元素交换
        2. 物理剔除：此时，曾经的王已经被流放到了数组最后。调用vector::pop_back()将其彻底从内存中抹除
        3. 向下调整(SiftDown/源码中的std::pop_heap):此时坐在堆顶的是曾经的小弟，它大概率德不配位。将其与左右孩子(2*i+1,2*+1)中较强的一个进行比较，如果不如孩子强，就发生交换，一路向下沉降，直到找到属于他的正确层级
    - 时间复杂度：O(logN)。最坏情况下，新上来的堆顶是个极弱的元素，会一路跌落到树底

## 3. 自定义排序规则(仿函数与算符重载)
1. 基础切换：大小顶堆的内置开关
    - <functional> 头文件中的 greater 和 less
2. 自定义对象的排序规则
    - 重载 < 运算符
        - 优先队列默认适用std::less<T>(即<运算符)来构建大顶堆
        - 如果我们希望cost越小的排在前面，我们需要欺骗一下STL：让<运算符在cost大的时候返回true
    - 编写仿函数
        - 在工业界我们更推崇仿函数(即重载了operator()的结构体或类)
