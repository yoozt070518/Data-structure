# 二叉树

## 1. 概念定义与数学性质

### 1.1 什么是二叉树
在数据结构中，二叉树最纯正的定义是**递归定义**

- 二叉树是n(n >= 0)个节点的有限集合。它要么是空集(n==0),要么由一个根节点加上两棵树分别成为左子树和右子树的，互不交叠的二叉树组成

- **注意**：二叉树**不是**有序树

- 核心铁律：二叉树的左右子树次序不能颠倒。即使某个节点只有一个子节点，你也必须明确指出它是“左孩子”还是“右孩子”。在底层代码实现时，这是left指针还是right指针的原则问题

### 1.2 两大特殊形态：满二叉树vs完全二叉树

1. 满二叉树
在国内教科书的语境下，满二叉树的定义非常极端：一颗高度为h的二叉树，如果每一层的节点数都达到了最大值，也就是说没有一个地方时空着的，那它就是满二叉树

- 特点：除了叶子节点，所有节点的度都必须是2

- 数学特征：如果高度为h(根节点算第一层)，那么它的节点总数必然是
            N = 2^h - 1

2. 完全二叉树
完全二叉树是满二叉树的“为完全体”。它的定义是：一颗高度为h的二叉树，除去第h层(最底层)外，其余各层(1 ~ h-1层)的节点数都达到最大值。且第h层的所有节点都连续集中在最左边，绝不能留有空隙

**辨析**
满二叉树**一定是**完全二叉树
但完全二叉树**不一定是**满二叉树 -- 它允许最后一层不满，但最后一层的节点必须想排队一样紧紧向左看齐，中间不能插空

**为什么要严苛地定义完全二叉树？**
因为只有当节点严格连续，不留空隙时，我们才能用连续的数组去完美映射这棵树地父子关系，而不需要浪费任何一个数组位置(不会产生内存碎片)。

### 1.3 核心数学性质
我们统一规定：根节点所在的层为第一层，树地高度为h

1. 性质一：每层地最大节点数
   - **结论**：在二叉树地第i层上，最多有2^(i-1)个节点
   - **理解**：第一层是根节点2^0 = 1个；第二层最多2^1 = 2个；以此类推，每往下走一层，由于是二叉分裂，最大节点数直接翻倍

2. 性质二：整棵树的最大节点数
    - **结论**：高度为h的二叉树，最多有2^h-1个节点
    - **推导**：把每层的最大节点数做一次等比数列求和

3. 性质三：度数关系
    - **结论**：对任何一颗二叉树，如果其叶子节点(度为0)的个数为n0，度为2的节点个数为n2则必然满足：
                    n0 = n2 + 1

### 2.2 如何释放一棵树
- 遍历析构(后序遍历初体验)
要销毁一棵树， 底层核心逻辑必须是：**先摧毁左子树，再摧毁右子树，最后安全地释放根节点自己**
```cpp
template <typename T>
class BinaryTree {
private:
    TreeNode<T>* root; // 树的根节点

    // 私有递归辅助函数：用于销毁以 node 为根的子树
    void clear(TreeNode<T>* node) {
        if (node == nullptr) {
            return; // 递归基（Base Case）：空节点不需要释放
        }

        // 1. 递归释放左子树
        clear(node->left);
        
        // 2. 递归释放右子树
        clear(node->right);

        // 3. 左右子树都安全释放了，此时释放当前节点
        // std::cout << "Debug: 正在释放节点 " << node->data << std::endl; // 方便你 Debug 观察
        delete node;
    }

public:
    BinaryTree() : root(nullptr) {}

    // 析构函数：对外接口，内部调用递归销毁
    ~BinaryTree() {
        clear(root);
        root = nullptr; // 彻底置空，防止二次析构
    }

    // 临时提供一个手动构建树的方法，供后续测试遍历使用
    void setRoot(TreeNode<T>* node) {
        root = node;
    }
};
```
### 3.层序遍历
层序遍历的逻辑非常符合人类的直觉：从根节点开始，一层一层地从上往下，从左往右顺次访问每一个节点。

我们再做DFS(深度搜寻)时，为了“留住回头路”，用的是后进先出的栈。
二层序遍历的要求是“先看到的层先处理，后看到的层后处理”，这是一种绝对的先来后到。因此，底层的核心支撑结构必须是先进先出的**队列**

- 代码实现
```cpp
template<typename T
void levelOrder(TreeNode<T>* root){
    if(root == nullptr) return;
    std::queue<TreeNode<T>*> myQueue;
    myQueue.push(root);

    while(!myQueue.empty()){
        TreeNode<T>* curr = myQueue.front();
        myQueue.pop();
        std::cout << curr->data << " ";
        if(curr->left != nullptr){
            myQueue.push(curr->left);
        }
        if(curr->right != nullptr){
            myQueue.push(curr->right);
        }
    }
}
