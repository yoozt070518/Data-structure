# 音视频 × AI × 高性能 — 长期学习路线

> 为你（大一 CS，目标音视频开发 + 拥抱 AI + 高薪）定制。
> 时长按你的真实节奏估算：**约 3-4 个月吃透一个大主题块**，且偏稳不偏激进。

---

## 〇、先读这段：怎么用这份清单

1. **这是几年的长跑，不是冲刺。** 下面整条路按你的节奏铺下来约 **3-4 年**，正好覆盖你大学剩下的时间并往后延一点。别被总量吓到——你现在大一就想清楚方向，已经赢在起点。
2. **阶段不必严格串行。** 标号是大致顺序，但音视频核心（阶段三）的中后期可以**并行**启动 AI（阶段四）的一部分；实战项目（阶段六）从阶段三就该开始边学边做，不要攒到最后。
3. **分清主次：**
   - **地基（阶段 0-2）**：必须扎实，是一切的入场券，省不得。
   - **本行核心（阶段 3）**：你的立身之本。
   - **高薪差异化（阶段 4-5）**：AI 工程 + CUDA/性能，是把你和"普通音视频工程师"拉开的护城河。
4. **沿用你的学习法**：每块都按 `原理 → 实现 → API/工具 → 练习` 走。下面每阶段我都标了「学什么」和「动手产出」，产出就是你的练习与作品集。
5. **时长是估计，不是 deadline。** 学快了往前赶，学慢了别自责，按实际调整。

## 阶段 0：DSA 收尾（图 + 动态规划）

**目标**：补上数据结构与算法最后两块硬骨头，建立完整算法基本盘（也是技术面试的核心）。

**学什么**
- **图论**
  - 原理：图的表示（邻接表/邻接矩阵）、DFS/BFS
  - 最短路：Dijkstra、Bellman-Ford、Floyd
  - 最小生成树：Prim、Kruskal（并查集）
  - 拓扑排序、连通分量
- **动态规划**
  - 原理：状态、转移方程、最优子结构、重叠子问题
  - 经典模型：背包、最长公共/上升子序列、区间 DP、树形 DP
  - 记忆化搜索 ↔ 递推的互相转化

**动手产出**
- LeetCode 刷题：图 + DP 各专题各刷 30-50 题，建立手感
- 把红黑树/AVL 的实现整理进你的"数据结构代码库"

**时长**：~4 个月　**资源方向**：算法导论（CLRS）相关章节、LeetCode 分类题单

---

## 阶段 1：现代 C++ + 多线程 + Linux 工具链

**目标**：从"会写 C++"升级到"能写工程级、高性能、并发的 C++"，并能在 Linux 下流畅开发。这是音视频底层和高性能的真正入场券。

**学什么**
- **C++ 面向对象与内存（你原计划的 Month 1）**
  - 构造/析构、拷贝控制（拷贝/移动构造、赋值运算符）、Rule of Three/Five
  - 多态、虚函数、vtable 原理
  - RAII、对象生命周期
- **现代 C++（Month 2）**
  - 移动语义、右值引用、完美转发
  - 智能指针（unique_ptr / shared_ptr / weak_ptr）及其原理
  - 模板、泛型编程、模板元编程入门
  - STL 深入（容器底层、迭代器、算法、lambda）
  - C++11/14/17/20 关键特性
- **多线程与并发（Month 3 的并发部分）**
  - 线程、mutex、条件变量、原子操作、内存序入门
  - 线程池、异步（future/promise/async）
  - 并发的常见坑（数据竞争、死锁）
- **Linux 开发环境与工具链**
  - Linux 基本使用 + Shell
  - 构建：CMake、Makefile
  - 调试：GDB
  - 版本控制：Git（熟练）
  - 性能初探：基本的 profiling 概念

**动手产出**
- 用现代 C++ + 智能指针重写你之前的数据结构（体会 RAII / 移动语义）
- 写一个**线程池**（并发经典练手项目）
- 所有项目都用 CMake 管理、Git 托管、GDB 调过

**时长**：~6-8 个月（C++ 进阶一块 + 多线程/Linux 一块）
**资源方向**：现代 C++ 经典书 + cppreference 官方文档；多线程可配合《C++ Concurrency in Action》一类

## 阶段 2：操作系统 + 计算机网络（+ 体系结构基础）

**目标**：补齐系统层认知。音视频是"贴着系统跑"的领域（IO、内存、网络），性能优化更要懂硬件。这一阶段决定你能走多深。

**学什么**
- **操作系统**
  - 进程 / 线程 / 调度
  - 内存管理（虚拟内存、分页）
  - 文件系统、IO（阻塞/非阻塞、IO 多路复用 epoll —— 流媒体服务器要）
  - 同步原语（信号量、互斥）
- **计算机网络**
  - TCP/IP 模型、TCP vs UDP（音视频实时传输偏 UDP）
  - HTTP/HTTPS、HTTP/2、QUIC
  - 套接字编程（socket）
  - 流媒体相关：为后面的 RTMP/WebRTC 打底
- **计算机组成 / 体系结构（够用即可，为性能优化）**
  - CPU、Cache 与内存层级（局部性原理 —— 优化的根）
  - 流水线、SIMD 概念

**动手产出**
- 写一个简单的 **TCP/UDP 网络程序**（如聊天室、文件传输）
- 用 epoll 写一个**小型并发服务器**
- 这两个项目是后面流媒体服务器的雏形

**时长**：~5-7 个月
**资源方向**：**CSAPP《深入理解计算机系统》**（系统这块的圣经，强烈推荐通读）；《计算机网络：自顶向下方法》

## 阶段 3：音视频专业核心 ★ 本行立身之本

**目标**：真正进入音视频领域。从原理到 FFmpeg 到实时通信，建立完整的音视频工程能力，并产出第一批硬核作品。

**学什么**
- **多媒体基础**
  - 数字音频：采样、量化、PCM、声道
  - 数字视频：帧、分辨率、帧率、色彩空间（YUV / RGB、为什么用 YUV）
  - 时间戳、音视频同步（PTS/DTS）
- **编解码原理**
  - 视频编码：H.264 / H.265(HEVC) / AV1（帧内/帧间预测、变换量化、熵编码的大致原理）
  - 音频编码：AAC、Opus
  - 码率控制、GOP 结构
- **封装与协议**
  - 封装格式：MP4、FLV、TS、MKV
  - 流媒体协议：RTMP、HLS、DASH（点播/直播分发）
- **FFmpeg（重点，往底层钻）**
  - 架构与核心库（libavcodec / libavformat / libavfilter / libswscale）
  - 命令行熟练 + C API 编程
  - 自己写：解封装、解码、编码、转码、滤镜处理
  - 读源码（挑核心模块）
- **渲染与播放（SDL 在这里正式用上）**
  - SDL：音视频渲染、事件循环
  - OpenGL 基础（视频渲染、后续接 AI 滤镜要）
- **WebRTC（实时通信 —— 当下高薪热点）**
  - 实时通信原理、信令、NAT 穿透、传输（RTP/RTCP）
  - 在直播、会议、云游戏中的应用

**动手产出（这些是作品集的核心）**
- **自己写一个视频播放器**（FFmpeg 解码 + SDL 渲染 + 音视频同步）——音视频入门的"hello world 级"硬核项目
- 写一个**转码工具**（用 FFmpeg API，支持格式/分辨率/码率转换）
- 一个**简单的直播推拉流 demo**（RTMP 或基于 WebRTC）

**时长**：~8-12 个月（编解码 + FFmpeg 一大块，WebRTC + 渲染一块）
**资源方向**：**FFmpeg 官方文档**；中文音视频经典博客（雷霄骅的系列是绕不开的入门资料）；编解码找一本系统讲原理的书；WebRTC 官方文档 + 经典实践教程

## 阶段 4：AI 基础 + 工程化 ★ 差异化能力

**目标**：拿到"AI 含量"，但走**工程路线**——重点是把 AI 模型用进音视频管线、并能部署上线，而不是去做算法研究。

**学什么**
- **前置**
  - Python（若不熟，快速补；做 AI 绕不开）
  - 数学够用即可：线性代数、概率、微积分的核心概念（不必死磕证明）
- **深度学习基础（懂原理 + 会用，不求做研究）**
  - 神经网络、反向传播、训练流程
  - PyTorch（主流框架，重点）
  - CNN（视觉基础）、Transformer（多模态/大模型基础）
  - 扩散模型（Diffusion）概念（生成式视频/音频的核心）
- **AI 工程化（你的主战场）**
  - 模型推理流程、ONNX（模型交换格式）
  - 推理加速：TensorRT、模型量化、剪枝
  - 模型部署：服务化、端侧部署概念
- **AI × 音视频落地（把两条线接起来）**
  - 视频超分辨率（Real-ESRGAN 类）
  - 视频插帧（RIFE 类）
  - AI 降噪、视频修复
  - 视频理解（检测/分类/字幕）、实时 AI 处理（背景替换、实时字幕）

**动手产出**
- 用 PyTorch 跑通一个小模型（建立手感）
- **把一个开源视频增强模型（超分/插帧/降噪）部署起来、并接到你的 FFmpeg/播放器管线里**——这是"AI×音视频"的第一个真正交叉项目，含金量高
- 尝试用 ONNX/TensorRT 给它做推理加速

**时长**：~6-9 个月（DL 基础一块 + 工程化/落地一块）
**资源方向**：**《动手学深度学习》(d2l)** 这类偏实战的入门；PyTorch 官方教程；ONNX/TensorRT 官方文档；目标开源项目的 GitHub 仓库

## 阶段 5：CUDA / 高性能 / 推理优化 ★ 护城河（最抗替代、最高薪）

**目标**：拿下音视频和 AI **共同的**硬核高薪技能。会 GPU 优化的人少、值钱，且"优化 AI 的人"本身最不容易被 AI 取代。

**学什么**
- **性能优化基础**
  - Profiling（找瓶颈）、内存访问优化（呼应阶段 2 的 Cache 局部性）
  - SIMD（CPU 向量化）
  - 多线程性能调优
- **CUDA / GPU 编程（重点）**
  - GPU 架构与并行模型（线程、block、grid、warp）
  - CUDA C++ 编程、内存层次（global/shared/register）
  - 常见并行模式与优化技巧
  - GPU 加速音视频处理（解码、滤镜、AI 推理）
- **AI 推理优化深入**
  - TensorRT 深入、算子优化
  - 量化部署（INT8 等）、端侧/边缘部署
  - 高吞吐推理服务

**动手产出**
- 写几个 **CUDA kernel**（如图像处理、矩阵运算），体会 GPU 加速
- **给阶段 4 那个 AI 视频项目做 GPU 加速 + 推理优化**，量化前后性能对比
- 一个"**实时**"项目（如实时视频超分/降噪），逼自己做到低延迟

**时长**：~6-9 个月
**资源方向**：CUDA 官方文档 + 《CUDA C 编程权威指南》一类；TensorRT 官方文档与示例

## 阶段 6：整合项目 + 作品集 + 求职（全程穿插，最后冲刺）

**目标**：把三条线拧成拿得出手的整合项目，打磨作品集，准备求职/实习。

**做什么**
- **整合性大项目（瞄准那个高薪交叉，挑 1-2 个做深）**
  - 实时视频超分/降噪系统（音视频管线 + AI 模型 + GPU 加速 + 低延迟）
  - AI 直播增强（实时美颜/背景/字幕 + WebRTC）
  - 生成式视频处理管线（接生成模型 + 工程化 + 优化）
- **作品集 > 证书**：GitHub 把项目整理好，写清楚你解决了什么、怎么优化的、性能数据
- **开源贡献**：给 FFmpeg 或相关开源项目提 PR（极大加分）
- **求职准备**：算法面（阶段 0 的底子）+ 系统设计 + 音视频/AI 领域知识
- **实习节点**：通常大三可以开始投实习——别等"全学完"，有阶段 3 的项目 + 基础就可以试

**贯穿始终**
- **跟踪最新**：生成式视频/音频模型、行业招聘趋势，定期上网查（我的认知有截止，最新动态你要自己盯）
- 持续刷题保持算法手感

## 附：主次与避坑（再强调一遍）

**必须扎实（不可省）**：阶段 0-2 的地基。没有 C++/系统/网络的底子，音视频和高性能都是空中楼阁。

**你的护城河**：阶段 5 的 CUDA/性能 + 阶段 4 的 AI 工程化。这两样决定你是"高薪稀缺"还是"普通"。

**要警惕**：
- 别停在"只会调 FFmpeg/OpenCV API"——那是初级、价值在被压缩。要懂原理、能优化。
- 别只会"拿现成生成模型拼 demo"——没壁垒。要会部署、优化、接进真实管线。
- 别去硬卷"训练大模型的算法研究岗"——门槛极高、坑位少，对本科自学不是最现实的高薪路径（真想走得读研）。你的赢面在**工程化 + 优化**。
- "会用 AI 工具（Claude Code 等）提效"是基本功、人人都有，**不构成差异化**——差异化是上面那个三线交叉。

**一句总结**：照这条路走下来，你会成为一个**"懂音视频管线 + 能把 AI 模型部署优化上线 + 会 GPU/性能优化"**的工程师——这正是当下又稀缺又高薪的那个位置。
